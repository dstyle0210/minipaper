CSS("Cascading Style Sheets")란?

상속성(Cascading)

물처럼 흘러내려서 CSS?

상속성 제외

텍스트 관련 속성(text-align, font-size 등)은 상속

박스모델 관련 속성(width,height)는 영향은 줄수 있지만 상속안됨

우선순위(Priority Rules)

  • 1. "!important" 구문
  • 2. inline 직접선언 ( <E style=""></E> )
  • 3. id (#) 로 선언
  • 4. class (.) 로 선언
  • 5. 엘리먼트로 선언
  • 6. 유니버셜 (*) 선언
  • 7. 작성순서에 따른 오버라이딩 | 미디어쿼리에 따른 오버라이딩
  • 8. native style sheet (네이티브 스타일 시트) | 상속성에 따라 상속된 선언

오버라이딩

우선순위가 같다면, 나중에 선언된 속성이 적용됩니다.

See the Pen [CSS] 오버라이딩 by 원용봉 (@dstyle0210) on CodePen.

셀렉터(Selector) - Basic

  • Universal Selector (*)

    *{color:red;}

  • Element Selector (E)

    span{color:red;}

  • id Selector(#)

    #id{color:red;}

  • Class Selector(.)

    .class{background-color:gray;}

  • Group Selector(,)

    h1 , #id , .class{color:red; }

셀렉터(Selector) - Attribute

  • [attribute]

    [class]{color:red;}

  • [attribute=value]

    [class="test"]{color:red;}

  • [attribute|=value]

    [class|="test"]{color:red;}

  • [attribute^=value]

    [class^="test"]{color:red;}

  • [attribute~=value]

    [class~="test"]{color:red;}

  • [attribute*=value]

    [class*="test"]{color:red;}

  • [attribute$=value]

셀렉터(Selector) - Search

  • inside Selector ( ) // 빈칸

  • Child Selector(>)

  • After Selector(+)

    h1+span{width:500px;}

  • Preceded Selector(~)

    h1~span{width:300px}

가상(Pseudo) 셀렉터 란?

Pseudo Class(피도 클래스)라는것은 엘리먼트가 특정한 상태일때 을 지칭합니다.

Pseudo Element(피도 엘리먼트)는 가상의 엘리먼트를 생성해서 속성을 지정합니다.

단독으로 사용할수는 있으나, 추천하지는 않으므로, 언제나 셀렉터에 연결하여 선언됩니다.

컬러(Color)

  • 컬러이름.(red , blue 등)
  • RGB 또는 RGBA방식 == rgb(0,0,0,0)
  • HEX(Hexadecimal)값 방식 == #RRGGBB (RR == 00~FF)

단위(Unit)

단위(Unit) : 절대적인 단위

  • px : 픽셀(가장 기본적인 사용)
  • pc : 파이카(1pc == 12pt == 1/6in)
  • pt : 포인트(1/72in)
  • cm : 센티미터
  • mm : 밀리미터
  • in : 인치

단위(Unit) : 상대적인 단위

  • em : 부모의 font-size을 1em 으로 적용하며, 부모의 폰트사이즈가 늘어나면 함께 들어난다.
  • % : 부모의 width을 100%로 적용하며, 부모의 width가 늘어나면 함께 늘어난다.
  • ch : "0" (숫자 0) 의 가로값을 기반으로 적용되는 사이즈.
  • ex : "x" (영문 소문자 x) 의 높이값을 기반으로 적용되는 사이즈.
  • rem : root 즉 html의 font-size을 1rem으로 기반하여, 절대사이즈로 적용.
  • vw : window의 viewport width을 기준하여 절대 적용됩니다.(그냥 window width의 1/100)
  • vh : window의 viewport height 기준하여 절대 적용됩니다.(그냥 window height의 1/100)
  • vmin : window의 width 또는 height 중 작은 사이즈의 1/100
  • vmax : window의 width 또는 height 중 큰 사이즈의 1/100

블럭요소(Block)

박스모델(Box-Model)

  • Content : 내부의 내용을 말합니다. 물론 다른 엘리먼트가 들어올수도 있습니다.
  • Padding : 컨덴츠와 보더사이의 간격입니다. 배경의 영향을 받습니다.
  • border : 컨덴츠와 패딩을 포함한 영역의 내곽선입니다. 가로값의 영향을 받습니다.
  • Margin : 보더와 외부 엘리먼트의 간격입니다. 배경의 영향을 받지않습니다.
  • outline : 컨덴츠와 패딩 및 보더를 포함한 포함한 외곽선입니다. 가로값의 영향을 받지 않습니다.

디스플레이(Display)

  • 기본값 : 엘리먼트형태 상속
  • 자식상속 : 안함
  • 작성방법 : display: inline | block | inline-block | none | flex | table | table-cell;
값 설명
inline 인라인 조건을 가진 엘리먼트로 적용합니다.
block 블럭 조건을 가진 엘리먼트로 적용합니다.
inline-block 인라인-블럭 조건을 가진 엘리먼트로 적용합니다.
none 엘리먼트 안보이도록 처리합니다.
flex CSS3의 flex 레이아웃 구조의 컨테이너로 적용합니다. (기본은 block속성을 따라갑니다.)
table "table" 엘리먼트의 조건으로 적용합니다. (기본은 inline-block을 따라갑니다.)
table-cell "td" 또는 "th" 엘리먼트의 조건으로 적용합니다. (기본은 inline-block을 따라갑니다.)

값 설명
visible 화면상 보여지도록 합니다.
hidden 화면상 보여지지 않도록 합니다.

인라인(텍스트) 요소

인라인 박스모델

폰트명(font-family)

  • 기본값 : 브라우저의 기본폰트
  • 상속성 : 있음
  • 작성방법 : font-family: (generic-family | family-name)+|initial|inherit;

제너릭명?

  • Serif(쉐리프) : 곡선을 주로 사용하여 작성된 폰트 스타일들 (예 : Times New Roman , Georgia)
  • Sans-serif(산쉐리프) : 직선을 주로 사용하여 작성된 폰트 스타일들 (예 : Arial , Verdana)
  • Monospace(모노스페이스) : 곡선과 직선을 혼합하여 작성된 폰트 스타일들 (예 : Courier New , Lucida Console)

색상(Color)

  • 기본값 : 브라우저 기본컬러
  • 상속성 : 있음
  • 작성방법 : color: HEX(#RRGGBB) | String | RGBA | initial | inherit;

크기(font-size)

  • 기본값 : medium
  • 상속성 : 있음
  • 작성방법 : font-size:medium|xx-small|x-small|small|large|x-large|xx-large|smaller|larger|length|initial|inherit;

굵기(font-weight)

  • 기본값 : 부모 상속됨
  • 자식상속 : 상속됨
  • 작성방법 : font-weight: normal|bold|bolder|lighter|number;

좌우정렬(text-align)

  • 기본값 : 부모상속됨
  • 자식상속 : 상속됨
  • 작성방법 : text-align: left|right|center|justify;

상하정렬(vertical-align)

  • 기본값 : "baseline"
  • 자식상속 : 안됨
  • 작성방법 : vertical-align: baseline | length | sub | super | top | text-top | middle | bottom | text-bottom;

글자간격, 자간(letter-spacing)

  • 기본값 : 부모상속
  • 자식상속 : 상속
  • 작성방법 : letter-spacing: normal|length;

들여쓰기(text-indent)

  • 기본값 : 0
  • 자식상속 : 안됨
  • 작성방법 : text-indent: length;

데코레이션(Decoration)

  • 기본값 : 부모상속됨
  • 자식상속 : 상속됨
  • 작성방법 : text-decoration: none|underline|overline|line-through;

텍스트 변형(text-transform)

  • 기본값 : 부모상속됨
  • 자식상속 : 상속됨
  • 작성방법 : text-transform: none|capitalize|uppercase|lowercase;

그림자(text-shadow)

  • 기본값 : none
  • 상속성 : 있음
  • 작성방법 : text-shadow: h-shadow v-shadow blur-radius color|none;

  • 기본값 : 부모상속됨
  • 자식상속 : 상속됨
  • 작성방법 : white-space: normal|nowrap|pre|pre-line|pre-wrap;

인라인높이, 자평(line-height)

  • 기본값 : 부모상속
  • 자식상속 : 상속
  • 작성방법 : line-height: normal|number|length;

단어 간격(word-spacing)

  • 기본값 : 부모상속
  • 자식상속 : 상속
  • 작성방법 : word-spacing: normal|length;

단어 줄바꿈 방식(word-break)

CJK(중국어,일본어,한국어)가 "아닌" 언어을 기준하여 줄바꿈 기준을 적용

  • 기본값 : 부모상속됨
  • 자식상속 : 상속됨
  • 작성방법 : word-break: normal|break-all|keep-all

수업 정리

CSS 란?

  • 상속성(Cascading) : 부모 또는 루트(:root)의 속성을 상속 받는 언어.
  • 인라인(텍스트)는 대부분 상속 , 블럭는 대부분 상속안됨

우선순위(Priority Rules)

  • 1. "!important" 구문
  • 2. inline 직접선언 ( <E style=""></E> )
  • 3. id (#) 로 선언
  • 4. class (.) 로 선언
  • 5. 엘리먼트로 선언
  • 6. 유니버셜 (*) 선언
  • 7. 작성순서에 따른 오버라이딩
  • 8. native style sheet (네이티브 스타일 시트) | 상속성에 따라 상속된 선언

작성법

적용방법

  • "link"엘리먼트

    <link rel="stylesheets" href="CSS경로" />

  • "style"엘리먼트

    <style>
    /* CSS 코드 */
    </style>

  • inline 직접선언

    <span style="스타일코드"></span>

셀렉터(Selector)

  • Universal Selector (*) : 전체선택

    *{color:red;}

  • Element Selector (E) : 엘리먼트선택

    span{color:red;}

  • id Selector(#) : id선택

    #id{color:red;}

  • Class Selector(.) : class선택

    .class{background-color:gray;}

  • Group Selector(,) : 그룹핑선택

    h1 , #id , .class{color:red; }

  • [attribute] : 어트리뷰트가 있으면 선택

    [class]{color:red;}

  • [attribute=value] : 어트리뷰트 값이 같으면 선택

    [class="test"]{color:red;}

  • [attribute^=value] : 어트리뷰트가 값으로 시작하면 선택

    [class^="test"]{color:red;}

  • inside Selector ( ) : 하위요소 검색

    h1 span{border:1px solid #000;}

  • Child Selector(>) : 자식요소 검색

    h1>span{background-color:gray}

  • After Selector(+) : 다음요소 검색

    h1+span{width:500px;}

가상 셀렉터(Selector)

  • Selector:checked : 체크되면 선택

    input[class=checkbox]:checked{}

  • Selector:disabled : 비활성화되면 선택

    input[class=checkbox]:disabled{}

  • Selector:focus : 포커스되면 선택

    input:focus{color:red;}

  • Selector:hover : 마우스가 올라가면 선택

    a:hover{}

  • Selector:first-child : 첫번째 자식요소 선택

    div p:first-child{}

  • Selector:nth-child(number | odd | even | n) : 자식들의 순서로 선택

    div p:nth-child(2){}

  • Selector:read-only : 읽기전용일때 선택

    input:read-only{}

  • Selector::before or Selector:before : 가상요소를 앞에 생성하여 선택

    .clearfix::before{}

  • Selector::after or Selector:after : 가상요소를 뒤에 생성하여 선택

    .clearfix::after{}

컬러(color)와 단위(unit)

  • 컬러는 RGBA 방식과 HEX(#RRGGBB)이 주로 쓰인다.
  • 단위는 "px" , "%" , "em" , "rem" 이 주로쓰인다.

박스모델 , 인라인박스모델

  • Box-Model
  • Inline-Model

박스사이징(Box Sizing)

  • 디스플레이(Display)

    디스플레이는 요소의 특성을 바꾼다.

    주속성 : none , block , inline , inline-block , flex

  • 비지빌리티(visibility)

    비지빌리티는 요소가 보이는가 안보이는가만 바꾼다.

    주속성 : hidden , visible

가로값(width) , 세로값(height)

  • 대부분은 "px","%","em","rem"을 쓴다.
  • 앞에 "min-" 이나 "max-"을 붙일수 있다.
  • "box-sizing" 여부에 따라 범위가 틀려진다.

  • 패딩(padding)

    배경을 영향을 받는 border-edge 내부 영역.

    요소들 끼리 겹치는 경우는 없다.

  • 마진(margin)

    배경의 영향을 안받는 border-edge 외부 영역

    요소들 끼리 겹치는 경우가 있다.

  • 보더(border)

    컨덴츠와, 패딩 이후의 내곽선

    border-radius라는 속성으로 둥글게 만들수도 있다.

    width값에 영향을 준다.

    요소들 끼리 겹치는 경우는 없다.

  • 아웃라인(outline)

    보더 밖으로 선언되는 외곽선

    outline-offset 이라는 속성으로 더 멀리 보낼수 있다.

    width값에 영향을 안준다.

    요소들 끼리 겹치는 경우가 있다.

오버플로우(Overflow)

컨덴츠가 보이는 영역에서 벗어날때 처리

scoll-x , scroll-y 같은 식으로 가로,세로도 별도 가능.

폰트 속성들

  • 폰트명(font-family) : 사용될 폰트명 또는 제너릭명으로 적용합니다.
  • 색상(Color) : 글자의 색상을 지정합니다.
  • 크기(size) : 글자의 크기를 지정합니다.
  • 굵기(weight) : 글자의 굵기를 지정합니다. (bold == 700)

  • 자평,텍스트높이(line-height)

    인라인 요소 중 한줄에 대한 높이를 지정한다.

  • 자간(letter-spacing)

    글자와 글자사이의 간격에 영향을 준다.

  • 좌우정렬(text-align)

    인라인에만 적용된다.

    요소의 내부 컨덴츠에 적용된다.

    상속되며, 절대적이다.

  • 상하정렬(vertical-align)

    인라인에만 적용된다.

    요소 자체의 정렬에 적용된다.

    상속안되고, 상대적이다.

  • 줄바꿈 방식(white-space)

    줄바꿈에 대한 처리를 기준한다.

    요소 내의 컨덴츠가 요소의 영역을 넘어갈때만
    쓰는건 아니다.

  • 단어 줄바꿈 방식(word-break)

    CJK언어가 아닌 언어에만 적용된다.

    단어를 기준하여 줄바꿈을 처리한다.
